# Page Specs

Each file in this directory is a frontend implementation contract for one route.

Before coding a page:

1. Open the relevant spec.
2. Confirm it is detailed enough.
3. Update it if behaviour, data, layout, or acceptance criteria need to change.
4. Then implement code against the spec.
